我的AstroNvim配置

阅读时间 5 分钟 · 修改于 6 天前
neovimIDE

# 写在前面

Neovim是大家公认的、最好用的一款TUI编辑器，其中必须给到夯的是Neovim极其优秀的、可扩展性极高的插件系统，如果运用得炉火纯青，我们就能创造出无限可能。

但是，一堆文件堆在配置文件夹里，很难管理，怎么办？

AstroNvim登场了！

轮椅一般的，很美妙啊，是吧？

接下来，我将给出我的AstroNvim配置文件。当然，如果你也想用，可以克隆我的Github仓库，随时欢迎。

NOTE

「我自己都没配置好呢。我会边写文章边配置的，只需坐和放宽。」

# 更新日志

2026·06·19 - 加入了Sidekick插件，完善AI编辑器的功能，还有一堆微调。

# 安装AstroNvim（Windows 10+）

参考网站：https://docs.astronvim.com/

# 先决条件

我们需要：

  • 一款Nerd Font。我首推Jetbrains系列的字体，当然人各有所好，如果你想选其它的也不是不行。
  • 安装Neovim v0.11+。打开终端，运行winget install neovim
  • 一款C编译器。这是最难受的一步 （至少对我来说）。先前往Visual Studio官网，在页面下面找到 「Visual Studio 2026 生成工具」，下载之后安装，功能看着选就好。
  • Optional：一些实用工具的安装。打开终端，运行winget install ripgrep OpenJS.NodeJS Python.Python.3.14

NOTE

「我说winget也是轮椅，有没有懂的」

# 脚本安装

到了最激动人心的时刻了，记得先保证与Github的连接稳定。大家都知道的，我就不多说了，都到这一步了肯定都有吧。

  1. 自己先备份好之前的Neovim配置文件
  2. 复制并运行以下命令
bash 
git clone --depth 1 https://github.com/AstroNvim/template ~/.config/nvim
rm -rf ~/.config/nvim/.git
nvim
  1. 等插件都安装好，等待的同时你可以冲杯咖啡慢慢喝

不出意外的话，到这里就已经安装好AstroNvim了。有想法的可以关掉这篇blog了；如果你是刚开始用AstroNvim的，也可以先关掉这篇blog，自己探索一下里面的功能，再进行配置。

# 配置

TIP

在这一步之前，记得先把配置文件中的以下行删去。

if true then return {} end -- WARN: REMOVE THIS LINE TO ACTIVATE THIS FILE

# Step 1: AstroCommunity

打开/lua/plugins/community.lua文件，这一章节我们都会在这个文件里作修改。

先前往AstroCommunity仓库，在里面挑选一些pack，然后以{ import = "astrocommunity.pack.lua" },的格式插入到return函数内。

以下是我的Pack配置，更新完之后记得重启nvim以下载插件。

lua 
---@type LazySpec
return {
	"AstroNvim/astrocommunity",
	-- 主题包
	{ import = "astrocommunity.colorscheme.catppuccin" },
	{ import = "astrocommunity.colorscheme.onedarkpro-nvim" },
	{ import = "astrocommunity.colorscheme.tokyodark-nvim" },

	-- Blink.cmp 自动补全
	{ import = "astrocommunity.completion.blink-cmp" },

	-- 语言包
	{ import = "astrocommunity.pack.cpp" },
	{ import = "astrocommunity.pack.java" },
	{ import = "astrocommunity.pack.python" },
	{ import = "astrocommunity.pack.lua" },

	-- AI 配置
	{ import = "astrocommunity.ai.sidekick-nvim" },
}

在这一步，我们只需要把主题换一下。

# Step 3: AstroCore

打开/lua/plugins/astrocore.lua并新建/lua/plugins/keymaps.lua文件，这一章节我们都会在这两个文件里作修改。

NOTE

为什么要新建keymaps.lua文件呢？因为这样管理快捷键会方便很多。

astrocore.lua文件看起来很乱，我们可以先把那些注释还有示例代码删掉，再开始配置。

TIP

「配置以后记得补上注释，不然以后你也看不懂配置了哈」

astrocore.lua文件内容如下。

lua 
---@type LazySpec
return {
	"AstroNvim/astrocore",
	---@type AstroCoreOpts
	opts = {
		features = {
			large_buf = { size = 1024 * 256, lines = 10000 },
			autopairs = true,
			cmp = true,
			diagnostics = { virtual_text = true, virtual_lines = false },
			highlighturl = true,
			notifications = true,
		},
		-- Diagnostics configuration (for vim.diagnostics.config({...})) when diagnostics are on
		diagnostics = {
			virtual_text = false,
			underline = false,
		},
		filetypes = {},
		options = {
			opt = {
				-- vim.opt.<key>
				relativenumber = false,
				number = true,
				spell = false,
				signcolumn = "yes",
				wrap = true,
				scrolloff = 9,
				cursorline = true,
			},
			g = {
				-- vim.g.<key>
			},
		},
	},
}

keymaps.lua 文件内容如下。

lua 
---@type LazySpec
return {
	"AstroNvim/astrocore",
	---@type AstroCoreOpts
	opts = {
		mappings = {
			n = {
				-- navigate buffer tabs
				["<S-l>"] = {
					function()
						require("astrocore.buffer").nav(vim.v.count1)
					end,
					desc = "Next buffer",
				},
				["<S-h>"] = {
					function()
						require("astrocore.buffer").nav(-vim.v.count1)
					end,
					desc = "Previous buffer",
				},

				-- Sidekick settings
				["<leader>aa"] = {
					function()
						require("sidekick.cli").toggle("claude")
					end,
					desc = "Sidekick Toggle CLI",
				},
				["<leader>as"] = {
					function()
						require("sidekick.cli").select()
					end,
					-- Or to select only installed tools:
					-- require("sidekick.cli").select({ filter = { installed = true } })
					desc = "Select CLI",
				},
				["<leader>ad"] = {
					function()
						require("sidekick.cli").close()
					end,
					desc = "Detach a CLI Session",
				},
			},

			i = {
				-- Sidekick settings
				["<Tab>"] = {
					function()
						-- if there is a next edit, jump to it, otherwise apply it if any
						if not require("sidekick").nes_jump_or_apply() then
							return "<Tab>" -- fallback to normal tab
						end
					end,
					desc = "Goto/Apply Next Edit Suggestion",
				},
			},
		},
	},
}

有新的语言需求时，只需要前往Mason搜索并安装相应的lsp和dap即可。
